## Правила работы в репозитории

### 1. Ветка `main`
Ветка `main` всегда содержит только **стабильный, рабочий код**. Она должна быть готова к запуску в любой момент.  
Запрещено пушить в `main` напрямую, вносить туда сырые или недоделанные функции, а также оставлять сломанный код.

### 2. Работайте в своей веткк
Создайте себе личную, **отдельную от main ветку** и разрабатвайте в ней свои функции.  
Название ветки должно отражать суть задачи и быть понятным другим участникам. Обязательно указывайте в названии ветки того, кто над ней работает

### 3. Pull Request
Когда задача полностью готова и протестирована, откройте **Pull Request** в `main`.  
В описании PR обязательно укажите:
- **Что было сделано**

После того как ваши изменения будут проверены и **одобрены нами**, можно выполнить слияние (merge).

### 4. Перед началом работы необходима синхронизация
Прежде чем начать новый сеанс работы над проектом, обязательно **актуализируйте свою локальную копию** с удалённым репозиторием
